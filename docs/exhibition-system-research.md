# 展示会システム調査レポート

## 1. 対象スコープ

今回検討するシステムは以下の4つの機能領域：

| # | 機能領域 | 概要 |
|---|---------|------|
| 1 | 事前登録 | 来場者がWebフォームから展示会への来場を事前に登録 |
| 2 | 展示会当日の入場管理 | QRコード等による受付・入退場の記録・リアルタイム集計 |
| 3 | セミナー申し込み | 展示会内で開催されるセミナー・講演への事前申込・座席予約 |
| 4 | セミナー入場管理 | セミナー会場での受付・出席管理・定員管理 |

---

## 2. 展示会における事前登録の仕組み

### 2.1 一般的なフロー

```
来場者                          主催者側
  │                              │
  ├─ (1) Webフォームにアクセス ──→│
  │     氏名・会社名・役職       │
  │     メール・電話番号         │
  │     来場目的・興味カテゴリ   │
  │                              │
  ├─ (2) 確認メール受信 ←────────┤ 自動返信メール（QRコード付き）
  │                              │
  ├─ (3) リマインドメール ←──────┤ 開催数日前に自動送信
  │                              │
  └─ (4) 当日：QRコード提示 ──→ │ スキャンして入場処理
                                  │ → バッジ（入場証）印刷
```

### 2.2 事前登録で収集する主要情報

| 項目 | 必須/任意 | 備考 |
|------|----------|------|
| 氏名（漢字・カナ） | 必須 | バッジ印刷用 |
| メールアドレス | 必須 | QRコード送信先、連絡用 |
| 会社名 | 必須 | |
| 部署名 | 任意 | |
| 役職 | 任意 | マーケティング分析用 |
| 電話番号 | 任意 | |
| 郵便番号・住所 | 任意 | |
| 来場目的 | 任意 | 複数選択（情報収集/商談/技術調査等） |
| 興味のある出展カテゴリ | 任意 | 出展社への送客に活用 |
| 同意事項 | 必須 | 個人情報取扱い同意 |

### 2.3 登録種別

展示会によっては来場者を種別分けする：

- **一般来場者**: 通常の事前登録
- **招待者（VIP）**: 主催者・出展社からの招待コード付き登録
- **プレス（報道関係者）**: プレス専用登録フォーム
- **出展社関係者**: 出展社スタッフとしての登録
- **セミナー講演者**: 講演者として招待登録

## 3. 展示会当日の入場管理

### 3.1 受付方式の比較

| 方式 | メリット | デメリット | 採用例 |
|------|---------|-----------|--------|
| **QRコード** | 高速・非接触・コスト低 | スマホ依存 | 最も一般的 |
| **バーコード** | 印刷物対応可 | やや遅い | 招待状付き展示会 |
| **ICカード/NFC** | 超高速・再利用可 | カード配布コスト | 大規模国際展示会 |
| **顔認証** | 完全非接触 | 導入コスト高・プライバシー懸念 | セキュリティ重視の展示会 |
| **氏名検索（当日登録）** | 事前登録不要 | 低速・入力ミスのリスク | 事前登録なし来場者向け |

### 3.2 入場管理で必要な機能

1. **QRコード読み取り**: スマホ/タブレットのカメラ、または専用リーダーでスキャン
2. **来場者照合**: 事前登録データと突合し、本人確認
3. **バッジ印刷**: 名前・会社名・カテゴリ（色分け）をバッジに印刷
4. **入場記録**: タイムスタンプ付きで入場を記録
5. **リアルタイム集計**: 現在の来場者数・累計来場者数をダッシュボードで表示
6. **当日登録対応**: 事前登録していない来場者の当日受付
7. **再入場対応**: 同一来場者の再入場を識別（重複カウント防止）
8. **混雑状況モニタリング**: 時間帯別の入場者数推移

### 3.3 受付オペレーションのパターン

```
【パターンA: セルフ受付（大規模展示会向け）】
来場者 → QRコードをキオスク端末にかざす → バッジ自動印刷 → 入場

【パターンB: スタッフ受付（中小規模展示会向け）】
来場者 → QRコードをスタッフが読み取り → バッジ手渡し → 入場

【パターンC: 当日登録】
来場者 → タブレットで情報入力 → QRコード発行 → バッジ印刷 → 入場
```

## 4. セミナー申し込みの仕組み

### 4.1 展示会併催セミナーの特徴

展示会では、会期中に複数のセミナー・講演・ワークショップが開催されることが一般的。

- **基調講演（Keynote）**: 大ホール、数百〜数千人規模
- **出展社セミナー**: 小会場、数十〜数百人規模
- **ワークショップ**: ハンズオン形式、少人数
- **パネルディスカッション**: 複数講演者によるディスカッション

### 4.2 セミナー申し込みフロー

```
来場者                              主催者側
  │                                  │
  ├─ (1) セミナー一覧を閲覧 ──────→│ セミナー情報（日時/会場/定員/講演者）
  │                                  │
  ├─ (2) セミナー選択・申し込み ──→ │ 定員チェック → 座席確保
  │     ※事前登録済みなら情報連携   │
  │     ※複数セミナーまとめて申込可 │
  │                                  │
  ├─ (3) 確認メール受信 ←──────────┤ セミナーごとのQRコード or 統合QRコード
  │                                  │
  ├─ (4) 満席の場合 ←──────────────┤ キャンセル待ち登録
  │                                  │
  └─ (5) リマインド ←──────────────┤ セミナー前日/当日朝に通知
```

### 4.3 セミナー管理で必要な機能

| 機能 | 説明 |
|------|------|
| セミナー情報管理 | タイトル・概要・講演者・日時・会場・定員の登録 |
| タイムテーブル表示 | 日別・会場別のセミナースケジュール一覧 |
| 申し込み受付 | 定員管理付きのオンライン申し込み |
| キャンセル待ち | 満席時の自動キャンセル待ちリスト管理 |
| 時間帯重複チェック | 同一来場者が時間帯の重なるセミナーに申し込めないよう制御 |
| 申し込み変更・キャンセル | 来場者側からの変更・キャンセル操作 |
| リマインドメール | セミナー前の自動通知 |
| 空席状況表示 | リアルタイムの残席数表示 |

## 5. セミナー入場管理

### 5.1 セミナー受付フロー

```
来場者                          セミナー会場スタッフ
  │                              │
  ├─ QRコード提示 ──────────── → │ スキャンで出席記録
  │                              │ → 申込済み確認
  │                              │ → 座席案内（指定席の場合）
  │                              │
  │                              │ リアルタイムで出席率表示
  │                              │ No-show（欠席）の把握
  │                              │ 空席があれば当日申込受付
```

### 5.2 セミナー入場管理で必要な機能

1. **出席チェックイン**: QRコードスキャンで出席記録
2. **出席状況ダッシュボード**: 申込数 vs 実出席数のリアルタイム表示
3. **当日受付（ウォークイン）**: 空席がある場合の当日参加受付
4. **No-show管理**: 欠席者の把握と次回向けデータ蓄積
5. **座席管理（指定席の場合）**: 座席番号と来場者の紐づけ
6. **アンケート連携**: セミナー終了後のアンケート配布

## 6. 主要なデータエンティティ（概念レベル）

```
┌─────────────┐     ┌──────────────┐     ┌───────────────┐
│  Exhibition  │────<│ Registration │>────│    Visitor     │
│  (展示会)    │     │ (事前登録)    │     │  (来場者)      │
└──────┬──────┘     └──────────────┘     └───────┬───────┘
       │                                         │
       │ has many                                 │ applies for
       ▼                                         ▼
┌──────────────┐     ┌──────────────┐     ┌───────────────┐
│   Seminar    │────<│SeminarBooking│>────│    Visitor     │
│ (セミナー)    │     │(セミナー申込) │     │  (来場者)      │
└──────┬──────┘     └──────────────┘     └───────────────┘
       │
       │ held at
       ▼
┌──────────────┐
│    Venue     │
│  (会場/部屋)  │
└──────────────┘

┌──────────────┐
│  EntryLog    │  ← 展示会入場ログ & セミナー出席ログ
│ (入場記録)    │
└──────────────┘
```

### 主要エンティティ一覧

| エンティティ | 説明 | 主要属性 |
|-------------|------|---------|
| **Exhibition** | 展示会 | ID, 名称, 開催期間, 会場, 主催者 |
| **Visitor** | 来場者 | ID, 氏名, メール, 会社名, 部署, 役職, 電話 |
| **Registration** | 事前登録 | ID, 展示会ID, 来場者ID, 登録日時, 登録種別, QRコード, ステータス |
| **EntryLog** | 入場記録 | ID, 登録ID, 入場日時, 入場ゲート, 方式(QR/手動等) |
| **Seminar** | セミナー | ID, 展示会ID, タイトル, 概要, 講演者, 日時, 会場ID, 定員 |
| **SeminarBooking** | セミナー申込 | ID, セミナーID, 来場者ID, 申込日時, ステータス(確定/キャンセル待ち/キャンセル) |
| **SeminarAttendance** | セミナー出席 | ID, 申込ID, チェックイン日時 |
| **Venue** | 会場/部屋 | ID, 名称, 収容人数, 設備, フロア |
| **Exhibitor** | 出展社 | ID, 展示会ID, 会社名, ブース番号 |

## 7. 既存システム（市場調査）

### 7.1 代表的なサービス

| サービス | 特徴 | 価格帯 |
|---------|------|--------|
| **Pregis**（フリーダム・グループ） | 30年以上の展示会運営ノウハウ。事前登録〜データ分析まで | 要問合せ |
| **Q-PASS** | QRコード受付、出展社情報管理、フォーム作成 | 要問合せ |
| **シャノン** | リアル/オンライン対応、セミナー〜展示会まで一元管理 | 月額10万円〜 |
| **EXPOLINE** | 出展者管理機能も搭載、1万人超の大規模対応 | 要問合せ |
| **ECOS** | セミナー管理に強み、講演者管理・タイムテーブル | 要問合せ |
| **SPIRAL** | 定額制、QRコード・投票・セッション予約 | 月額5万円〜 |
| **CLOUD PASS** | 年間1,500件実績、チケット販売〜データ活用 | 要問合せ |
| **Spot-Recorder** | スマホのみで運用可、専用端末不要 | 月額2万円〜 |

### 7.2 共通する機能パターン

1. **Webフォーム作成** → カスタマイズ可能な登録フォーム
2. **QRコード自動発行** → メール送付
3. **当日受付** → QRコードスキャン + バッジ印刷
4. **リアルタイムダッシュボード** → 来場者数、属性分析
5. **CSV/Excelエクスポート** → データ出力
6. **メール自動送信** → 確認、リマインド、フォローアップ
7. **セミナー管理** → 申し込み、定員、出席管理
8. **データ分析** → 来場者属性、行動分析

## 8. 既存プロジェクトとの関連性

現在の「展示会ドキュメント一元管理システム」との関連：

| 観点 | ドキュメント管理システム | 来場者管理・セミナー管理 |
|------|----------------------|----------------------|
| 対象ユーザー | 出展社 → 主催者・協力会社 | 来場者 → 主催者 |
| 主要データ | 注文書・設計書（非構造化） | 来場者情報（構造化） |
| AI活用 | OCR + LLM で書類解析 | QRコード生成・読取 |
| 共通基盤 | 展示会マスター、出展社マスター | 展示会マスター |
| 技術スタック | Next.js + FastAPI + PostgreSQL | 同じスタックで拡張可能 |

**統合のメリット**: 展示会マスターや出展社マスターを共有でき、来場者データとドキュメント提出状況を横断的に分析可能。

## 9. システム検討にあたっての論点

### 優先度の判断ポイント

1. **まず事前登録 + 入場管理を作る** → 展示会運営の基本機能
2. **次にセミナー申込 + 入場管理を追加** → 事前登録の来場者データを活用
3. **既存ドキュメント管理システムとの統合** → 展示会マスターの共有

### 技術的な検討事項

- QRコード生成・読取ライブラリの選定
- バッジ印刷の仕組み（サーマルプリンタ連携 or PDF生成）
- リアルタイム集計のアーキテクチャ（WebSocket or ポーリング）
- メール送信基盤（SendGrid, SES等）
- 大量同時アクセスへの対応（展示会開場時のスパイク）
- オフライン対応（ネットワーク障害時の受付継続）
